## Overview

PromoMaster Pro can generate high-quality product images using Google's Gemini AI image generation model. This feature is perfect when you need professional product photography but don't have actual product images available.

## Function Signature

Here's the actual implementation from `geminiService.ts`:

```typescript
export const generateProductImage = async (
  productName: string
): Promise<string | null> => {
  try {
    const prompt = `A professional, high-quality commercial studio photography of ${productName}. Clean background, cinematic lighting, 4k resolution, hyper-realistic, professional product placement.`;
    
    const response = await ai.models.generateContent({
      model: 'gemini-2.5-flash-image',
      contents: {
        parts: [{ text: prompt }]
      },
      config: {
        imageConfig: {
          aspectRatio: "1:1"
        }
      }
    });

    // Iterate through parts to find the image data
    for (const part of response.candidates?.[0]?.content?.parts || []) {
      if (part.inlineData) {
        return `data:${part.inlineData.mimeType};base64,${part.inlineData.data}`;
      }
    }
    return null;
  } catch (error) {
    console.error("Error generating image:", error);
    return null;
  }
};
```

### In React Components

Here's how it's integrated in the `Management.tsx` component:

```typescript
const [isGeneratingImage, setIsGeneratingImage] = useState(false);

const handleGenerateImage = async () => {
  if (!productForm.name) {
    alert("Introduce un nombre de producto para generar una imagen.");
    return;
  }
  
  setIsGeneratingImage(true);
  const imageUrl = await generateProductImage(productForm.name);
  if (imageUrl) {
    setProductForm(prev => ({ ...prev, imageUrl }));
  }
  setIsGeneratingImage(false);
};
```

### With Error Handling

```typescript
const generateImage = async (productName: string) => {
  try {
    const imageUrl = await generateProductImage(productName);
    
    if (!imageUrl) {
      console.warn("Image generation returned null");
      // Use placeholder or default image
      return "/images/placeholder.png";
    }
    
    return imageUrl;
  } catch (error) {
    console.error("Image generation failed:", error);
    return "/images/error-placeholder.png";
  }
};
```

```text

professional product placement.
```

<Tip>

## Image Configuration

The function uses specific configuration for optimal results:

| Setting | Value | Purpose |
|---------|-------|----------|
| Model | `gemini-2.5-flash-image` | Fast, high-quality image generation |
| Aspect Ratio | `1:1` | Square format for product displays |
| Output Format | Base64 data URL | Direct embedding in HTML |

<Note>
  The 1:1 aspect ratio is ideal for product catalogs and advertising displays where consistent sizing is important.
</Note>

## Best Practices

### Use Descriptive Product Names

```typescript
// Good - Specific and descriptive
await generateProductImage("Red Delicious Apple");
await generateProductImage("Stainless Steel Coffee Maker");

// Less effective - Too generic
await generateProductImage("Fruit");
await generateProductImage("Appliance");
```

### Handle Null Returns

Always check for null returns and provide fallbacks:

```typescript
const imageUrl = await generateProductImage(productName);
const finalUrl = imageUrl || '/images/placeholder-product.png';
```

### Show Loading State

```typescript
const [generating, setGenerating] = useState(false);

const generate = async () => {
  setGenerating(true);
  const url = await generateProductImage(name);
  setImageUrl(url);
  setGenerating(false);
};

return (
  <button disabled={generating}>
    {generating ? 'Generating...' : 'Generate Image'}
  </button>
);
```

### Validate Before Generation

```typescript
if (!productName || productName.trim().length < 3) {
  alert("Please provide a valid product name");
  return;
}

const imageUrl = await generateProductImage(productName);
```

## Performance Considerations

<Warning>
  Image generation is more resource-intensive than text generation:
</Warning>

- **Response Time**: Typically 5-15 seconds per image
- **File Size**: Base64-encoded images can be large (100KB - 500KB)
- **Rate Limits**: Image generation may have stricter quotas
- **Bandwidth**: Consider storage solutions for production use

### Production Recommendations

```typescript
// Option 1: Save to cloud storage
const imageUrl = await generateProductImage(productName);
if (imageUrl) {
  // Upload to S3, Cloudinary, etc.
  const cloudUrl = await uploadToCloud(imageUrl);
  return cloudUrl;
}

// Option 2: Cache generated images
const cacheKey = `product-image-${productName}`;
const cached = localStorage.getItem(cacheKey);
if (cached) return cached;

const imageUrl = await generateProductImage(productName);
localStorage.setItem(cacheKey, imageUrl);
```

## Troubleshooting

### Image Not Generating

- Verify your API key is configured correctly
- Check if you have image generation quota remaining
- Ensure the product name is descriptive enough

### Poor Quality Results

- Make product names more specific
- Try adding category context: "Fresh Red Apple" vs "Apple"
- Experiment with different prompt keywords

### Large File Sizes

Base64 images can bloat your application:

```typescript
// Solution: Upload to cloud storage
import { uploadImage } from './cloudStorage';

const base64Url = await generateProductImage(productName);
if (base64Url) {
  const publicUrl = await uploadImage(base64Url, productName);
  return publicUrl;
}
```
