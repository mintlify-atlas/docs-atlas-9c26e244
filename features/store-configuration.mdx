## Overview

PromoMaster Pro supports multi-store operations, allowing you to manage separate locations with individual branding and information. Each store can be selected when launching displays, ensuring location-specific information appears on-screen.

## Store Data Structure

Each store location contains the following information:

```typescript
export interface Store {
  id: string;
  name: string;
  address: string;
  logoColor: string;  // Tailwind CSS class (e.g., 'bg-indigo-600')
}
```

<Note>
Stores use Tailwind CSS color classes rather than hex codes for consistent theming across the application.
</Note>

## Creating Store Locations

<Steps>
  <Step title="Access Store Management">
    Navigate to the Management section and select the **Sucursales** tab.
  </Step>
  
  <Step title="Enter Store Details">
    Fill in the store name and full address.
  </Step>
  
  <Step title="Choose Brand Color">
    Select a logo color from the available options (applied via Tailwind classes).
  </Step>
  
  <Step title="Save Location">
    Click **Guardar Local** to add the store to your system.
  </Step>
</Steps>

## Store Form Implementation

The store creation form is streamlined for quick data entry:

```tsx
<form onSubmit={e => {
  e.preventDefault();
  if (editingStoreId) { 
    onUpdateStore({...storeForm, id: editingStoreId}); 
    setEditingStoreId(null); 
  }
  else onAddStore(storeForm);
  setStoreForm({ name: '', address: '', logoColor: 'bg-indigo-600' });
}}>
  <div>
    <label>Nombre</label>
    <input 
      type="text" 
      required 
      value={storeForm.name} 
      onChange={e => setStoreForm({...storeForm, name: e.target.value})}
    />
  </div>
  <div>
    <label>Dirección</label>
    <input 
      type="text" 
      required 
      value={storeForm.address} 
      onChange={e => setStoreForm({...storeForm, address: e.target.value})}
    />
  </div>
  <button type="submit">Guardar Local</button>
</form>
```

## Multi-Store Management

### Store List Display

Stores are displayed in a card-based layout with visual branding:

```tsx
<div className="bg-white p-4 rounded-2xl border">
  <div className="flex items-center gap-4">
    <div className={`w-12 h-12 rounded-xl ${store.logoColor} flex items-center justify-center text-white font-bold text-xl`}>
      {store.name.charAt(0)}
    </div>
    <div>
      <h4 className="font-bold">{store.name}</h4>
      <p className="text-sm text-slate-500">{store.address}</p>
    </div>
  </div>
  <div className="flex gap-2">
    <button onClick={() => editStore(store)}>Edit</button>
    <button onClick={() => onDeleteStore(store.id)}>Delete</button>
  </div>
</div>
```

Each store card shows:
- Logo badge with first letter of store name
- Store name and full address
- Edit and delete actions on hover

## Store Selection in Dashboard

When creating playlists or launching displays, select the active store location:

```tsx
<div className="flex items-center gap-3">
  <label className="text-sm font-bold">Local:</label>
  <select 
    value={selectedStore}
    onChange={(e) => setSelectedStore(e.target.value)}
    className="bg-white border-2 rounded-xl px-4 py-2"
  >
    {stores.map(s => (
      <option key={s.id} value={s.id}>{s.name}</option>
    ))}
  </select>
</div>
```

<Tip>
The first store in your list is automatically selected as the default when you open the Dashboard.
</Tip>

### Announcement Display Integration

Store name appears at the top of announcement displays with an animated icon:

```tsx
<div className="mb-12 inline-flex items-center gap-6 bg-white/10 px-10 py-4 rounded-[2rem]">
  <Megaphone className="text-white animate-bounce" size={40} />
  <span className="text-2xl font-black text-white tracking-[0.4em] uppercase">
    {store.name}
  </span>

```

### Ticker Integration

The store name scrolls continuously in the bottom ticker:

```tsx
const tickerMessages = useMemo(() => {
  const messages = [
    product.name.toUpperCase(),
    "CALIDAD GARANTIZADA",
    "DISPONIBLE AHORA",
    store.name.toUpperCase()  // Store name in ticker
  ];
  return messages;
}, [currentItem, store.name]);
```

## Editing Store Information

Click the edit icon on any store card to update its information:

```tsx
const handleEditStore = (store: Store) => {
  setEditingStoreId(store.id);
  setStoreForm({...store});
};
```

The form switches to edit mode with an **Actualizar** button instead of **Crear**.

## Deleting Stores

<Warning>
Deleting a store is permanent. Ensure no displays are currently active using this store before deletion.
</Warning>

To delete a store:
1. Click the trash icon on the store card
2. Confirm the deletion
3. The store is immediately removed from the system

## Application State Integration

Stores are part of the global application state:

```typescript
export interface AppState {
  stores: Store[];
  products: Product[];
  announcements: Announcement[];
  currentStoreId?: string;     // Currently selected store
  currentItems?: DisplayItem[];
  displayDuration?: number; 
  view: 'dashboard' | 'display' | 'management';
}
```

The `currentStoreId` tracks which store is selected in the Dashboard for display purposes.

## Best Practices

<Accordion title="Complete Addresses">
Always include full, accurate addresses. This information is displayed prominently during product advertisements.
</Accordion>

<Accordion title="Consistent Naming">
Use clear, recognizable store names. Avoid abbreviations that customers might not understand.
</Accordion>

<Accordion title="Strategic Colors">
Choose logo colors that align with your brand identity and create visual distinction between locations.
</Accordion>

<Accordion title="Regular Updates">
Keep store information current, especially if locations change or relocate.
</Accordion>
